# Contributing to From Objects to Agents

Thank you for your interest in contributing to this book. This is an open academic project and we welcome contributions that improve its accuracy, clarity, and coverage.

## What we welcome

**Corrections** — factual errors, broken cross-references, incorrect pattern descriptions, outdated citations. Please open an issue describing the error and where it appears.

**Clarifications** — sections that are unclear, arguments that need strengthening, examples that could be more precise. Open an issue with your suggestion before making a pull request.

**New citations** — relevant papers, production case studies, or industry references that strengthen or challenge the arguments made. Include the full citation and a brief note on why it is relevant.

**Typos and prose** — small fixes welcome as direct pull requests without a prior issue.

## What we are not seeking at this stage

- New pattern entries. The fifteen-pattern selection is a deliberate theoretical claim and additions require significant discussion with the authors.
- Restructuring of existing chapters. The book's argument structure is intentional.
- Changes to the NLI canonical example. This example is drawn from active research and changes require author involvement.

## How to contribute

1. **Open an issue first** for anything beyond a typo fix. Describe the problem clearly and include the relevant section reference.
2. **Fork the repository** and create a branch named `fix/short-description` or `clarify/short-description`.
3. **Make your changes** — keep them focused. One issue per pull request.
4. **Submit a pull request** referencing the issue number. Describe what you changed and why.

## Style guidelines

The book uses a consistent prose style. Before contributing:

- No em dashes. Use a comma, semicolon, or restructure the sentence.
- No mid-sentence colons unless introducing a list.
- No "not only...but" constructions.
- Direct declarative constructions over hedged or passive ones.
- Section structure per pattern entry: GoF definition → translation argument → stochasticity delta → structural diagram → canonical example → composability.

## Citation format

References use APA style via the `references.bib` BibTeX file. New citations should be added to `references.bib` using the existing key format (`authorYYYYkeyword`) and cited inline as `[@key]`.

## Code of conduct

All contributors are expected to follow the [Code of Conduct](CODE_OF_CONDUCT.md).
